## Project Overview

ESP32 embedded application using ESP-IDF 5.2.1 and FreeRTOS. Target: ESP32 (Xtensa) on ESP32-WROVER-KIT 3.3V board. Implements LED, relay, button, hardware timer, software timer, and BLE peripheral functionality.

## Build & Flash Commands

Environment setup (run once per shell session):
```bash
export IDF_PATH=D:/esp32_sub/esp-idf-v5.2.1/esp-idf-v5.2.1
source $IDF_PATH/export.sh
```

```bash
idf.py build                        # Build project
idf.py -p COM13 flash               # Flash to device
idf.py -p COM13 monitor             # Serial monitor
idf.py -p COM13 flash monitor       # Flash and monitor combined
idf.py menuconfig                   # Configure build options (sdkconfig)
idf.py fullclean                    # Clean all build artifacts
```

VS Code alternative: use ESP-IDF extension commands ("ESP-IDF: Build/Flash/Monitor").

## Architecture

Three-layer design:

**BSP** (`main/bsp/`) ‚Äî GPIO pin definitions only:
- `BSP_LED_RED` = GPIO 19, `BSP_LED_GREEN` = GPIO 22
- `BSP_RELAY` = GPIO 23, `BSP_MCU_BUTTON` = GPIO 4

**Drivers** (`main/drivers/`) ‚Äî hardware abstraction:
- `led.c/.h` ‚Äî `led_on()`, `led_off()`, `led_toggle()`
- `button.c/.h` ‚Äî GPIO input + ISR
- `relay.c/.h` ‚Äî `relay_on()`, `relay_off()`
- `hwtimer.c/.h` ‚Äî ESP hardware timer setup

**Tasks** (`main/tasks/`) ‚Äî FreeRTOS application logic:
- `task_event.c/.h` ‚Äî central EventGroup for inter-task communication
- `task_led.c/.h` ‚Äî 100ms blink cycle
- `task_button.c/.h` ‚Äî button event detection
- `task_relay.c/.h` ‚Äî relay control
- `task_hwtimer.c/.h` ‚Äî hardware timer task
- `task_ble.c/.h` ‚Äî BLE peripheral with GATT attribute state machine
- `swtimer.c/.h` ‚Äî software timer utilities
- `console.c/.h` ‚Äî UART console (currently disabled in `app_main.c`)

**Entry point**: `main/app_main.c` initializes NVS flash, BSP GPIO, then creates all FreeRTOS tasks.

**Optional components** (`components/`) ‚Äî `cmd_system`, `cmd_wifi`, `cmd_nvs` are present but not enabled.

## Key Config Files

- `sdkconfig` ‚Äî auto-generated by `menuconfig`; do not edit manually
- `partitions.csv` ‚Äî flash layout: NVS (24KB), PHY (4KB), factory app (1.9MB), FAT storage (1MB)
- `.vscode/settings.json` ‚Äî ESP-IDF paths and COM13 port (Windows-specific)

## ESP-IDF Conventions

- Components declare their sources and dependencies in `CMakeLists.txt` via `idf_component_register(SRCS ... REQUIRES ...)`
- The main component requires: `nvs_flash`, `esp_event`, `esp_timer`, `bt`, `driver`
- BLE uses the ESP-IDF GATT attribute table pattern (index-based, see `task_ble.h`)
- `sdkconfig` changes require a rebuild; `idf.py fullclean` if switching targets
